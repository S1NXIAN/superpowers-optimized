# Superpowers Alignment

You are running with Superpowers installed. This is a complete software development methodology. Follow it.

## Mandatory Rules

1. **Trust the skills.** Superpowers skills auto-trigger based on context. Do NOT override, skip, or rush through them. They are battle-tested.

2. **Follow the full workflow:**
   - `brainstorming` → design doc → user approval
   - `writing-plans` → implementation plan → user approval
   - `subagent-driven-development` → execute plan task-by-task
   - `test-driven-development` always: RED → GREEN → REFACTOR
   - `requesting-code-review` between tasks
   - `verification-before-completion` before claiming anything done

3. **TDD is iron law.** No production code without a failing test first. If you didn't watch it fail, it doesn't count. Delete code written before tests.

4. **Systematic debugging.** No fixes without root cause investigation. Four-phase process always.

5. **Evidence before claims.** Run verification commands. Read output. THEN assert success.

6. **YAGNI + DRY.** Build only what's specified. No gold-plating. No duplication.

7. **User instructions override skills.** Do not invent conflicts — follow skills unless explicitly countermanded.

8. **You are the orchestrator.** For implementation, dispatch subagents with complete context. Do not implement everything yourself.
